# Timber & Trims

Handmade wooden eyeglass temples — hand-shaped, inlaid, and finished — fitted
to acetate (or full-wood) frame fronts. This repo is the marketing/portfolio
site for the brand, structured so it can grow into e-commerce later without a
rewrite.

## Tech stack & structure

- Next.js 16 (App Router) + TypeScript + React 19
- Tailwind CSS v4 — theme tokens defined in `app/globals.css` via `@theme`
  (there is no `tailwind.config.js`)
- Fonts loaded via `next/font/google` in `app/layout.tsx`: Fraunces (serif,
  headings) and Inter (sans, body)
- Path alias `@/*` → project root (see `tsconfig.json`)

Routes:

- `app/page.tsx` — homepage (hero, intro, signature pieces, process teaser,
  custom-order CTA)
- `app/gallery/page.tsx` — full product gallery with type/wood filters
  (`components/GalleryGrid.tsx`, client component)
- `app/gallery/[slug]/page.tsx` — individual product detail page
- `app/about/page.tsx` — maker's story
- `app/process/page.tsx` — "how it's made" walkthrough
- `app/contact/page.tsx` + `app/contact/actions.ts` — custom order / inquiry
  form (server action)

Shared UI lives in `components/`: `Header`, `Footer`, `ProductCard`,
`GalleryGrid`, `ContactForm`.

## Images

- All site images live under `public/images/`, grouped by section: `hero/`,
  `products/`, `process/`, `about/`.
- `data/image-manifest.json` records the `{ width, height }` of every
  processed image, generated by `scripts/process-images.mjs`. Don't edit it
  by hand.
- `lib/images.ts` exports `img(src, alt)`, which looks up dimensions from the
  manifest and returns a `ProductImage` (`{ src, alt, width, height }`).
  Always build image references through this helper so `next/image` gets
  correct intrinsic dimensions, and so a typo'd path fails at build time
  instead of at runtime.
- Original, full-resolution photos live in `source-images/` (gitignored, not
  deployed). To add new product photos: drop the originals in
  `source-images/`, add an entry to the `MAP` array in
  `scripts/process-images.mjs` (source filename, destination path under
  `public/images/`, max width), then run `node scripts/process-images.mjs` to
  generate the optimized JPEG and add its manifest entry.

## Design system

The palette is pulled directly from the wood and frame photography rather
than a generic template palette. Tokens are defined in `app/globals.css`
under `@theme`:

| Token | Hex | Use |
| --- | --- | --- |
| `cream` | `#faf3e8` | Page background |
| `cream-dark` | `#efe1cc` | Alternate section backgrounds, card chips |
| `paper` | `#fffdf8` | Card / image surface backgrounds |
| `walnut` | `#4a2f20` | Primary brand color — headings, footer, buttons |
| `walnut-light` | `#7a5436` | Hover state for walnut buttons |
| `oak` | `#c4915a` | Accent — CTAs, eyebrow labels, highlights |
| `oak-light` | `#e8d2ab` | Light accent backgrounds, tag chips, borders |
| `mahogany` | `#8a4332` | Secondary accent (e.g. form error text) |
| `forest` | `#3c5c4d` | Secondary accent (green) |
| `ink` | `#2b2118` | Body text |
| `ink-muted` | `#6f5f50` | Secondary / muted text |

Typography: `font-serif` (Fraunces) is used for all headings — `h1`-`h4` are
set to `font-serif` globally in `app/globals.css`. Body text and UI use
`font-sans` (Inter), the default.

Conventions used throughout the site:

- Section "eyebrow" labels: `font-serif text-sm uppercase tracking-[0.25em] text-oak`
- Page/section headings: `font-serif text-3xl sm:text-4xl font-semibold text-walnut`
- Body copy: `text-lg leading-relaxed text-ink-muted`
- Buttons: pill-shaped (`rounded-full`), solid `bg-walnut`/`bg-oak` with
  `text-cream`/`text-walnut`, uppercase tracked labels
- No dark mode — the warm cream background is the only theme

## Product data (and future e-commerce)

`lib/types.ts` defines the shapes:

- `ProductImage`: `{ src, alt, width, height }`
- `Product`: `slug`, `name`, `tagline`, `description`, `category`
  (`"eyeglasses" | "sunglasses"`), `wood`, `frameColor`, `accents: string[]`,
  `tags: string[]`, `images: ProductImage[]`, optional `featured`, and the
  **future e-commerce fields** `price?: number` and `available?: boolean`.

`data/products.ts` exports:

- `products: Product[]` — the full catalog
- `featuredProducts` — entries with `featured: true`, shown on the homepage
- `getProductBySlug(slug)` — used by `app/gallery/[slug]/page.tsx`

To add e-commerce later: populate `price` / `available` on each product, then
build cart/checkout on top of `products` and `getProductBySlug` — no changes
to the existing data shape should be needed.

## Adding new products, process steps, or pages

**New product:**

1. Process its photos (see "Images" above) so they land in
   `public/images/products/` and `data/image-manifest.json`.
2. Add an entry to `products` in `data/products.ts`, using `img()` for each
   image. Give it a unique `slug` — this becomes its URL at
   `/gallery/[slug]`.
3. Set `featured: true` if it should appear in the homepage "Signature
   pieces" section.

**New process step:** add an entry to `processSteps` in `data/process.ts`
(`title`, `description`, `image: img(...)`). The process page renders these
in order with an alternating image/text layout automatically.

**New page:** add a folder under `app/` with a `page.tsx`. Add a link in
`components/Header.tsx` (`NAV_LINKS`) and, if relevant,
`components/Footer.tsx` (`FOOTER_LINKS`).

## Contact form

`app/contact/actions.ts` contains a server action (`submitContactForm`) that
validates the form fields and currently just `console.log`s the submission —
**no email provider is wired up yet**. Before launch, connect it to a service
such as Resend or Formspree so messages actually reach an inbox.
