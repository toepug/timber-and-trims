# Timber & Trims

A showcase site for **Timber & Trims** — handmade wooden eyeglass temples,
hand-shaped and inlaid, then fitted to upcycled material (and full-wood) frame fronts.

The site is a portfolio/marketing site today (homepage, gallery, process,
about, and a contact form), built on a product data layer that's structured
so pricing, availability, and a cart/checkout flow can be layered in later
without a rewrite.

## Tech stack

- [Next.js 16](https://nextjs.org) (App Router, React 19, TypeScript)
- [Tailwind CSS v4](https://tailwindcss.com) (theme defined in `app/globals.css` via `@theme`)
- [`next/font`](https://nextjs.org/docs/app/building-your-application/optimizing/fonts) — Fraunces (headings) + Inter (body)
- [sharp](https://sharp.pixelplumbing.com) — one-time image processing (see `scripts/process-images.mjs`)

## Getting started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the site. The
page auto-updates as you edit files under `app/`.

Other scripts:

```bash
npm run build   # production build
npm run start   # run the production build locally
npm run lint    # eslint
```

## Project structure

```
app/                 Routes (App Router): home, gallery, gallery/[slug],
                     about, process, contact
components/          Shared UI (Header, Footer, ProductCard, GalleryGrid,
                     ContactForm, ...)
data/                Site content as structured data (products, process
                     steps, about page images)
lib/                 Types and small helpers (image lookup, contact form
                     constants)
public/images/       Optimized images served by the site, grouped by
                     section (hero/, products/, process/, about/)
data/image-manifest.json
                     Width/height for every image in public/images,
                     generated by scripts/process-images.mjs
source-images/       Original, full-resolution photos (gitignored — not
                     part of the deployed site)
scripts/             One-off build/content scripts
```

See [CLAUDE.md](./CLAUDE.md) for details on the design system, product data
shape, and conventions for adding new products, images, or pages.

## Deployment

This project deploys on [Vercel](https://vercel.com) — connect the GitHub
repo ([toepug/timber-and-trims](https://github.com/toepug/timber-and-trims))
and push to `main`. `vercel.json` pins the Framework Preset to `nextjs` so
the build always runs through the Next.js output adapter.
